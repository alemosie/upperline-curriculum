CSS Walkthrough

An Introduction to Cascading Style Sheets

What is CSS?

CSS stands for Cascading Style Sheets. We use CSS to apply style and formatting to websites. It's a best practice as a programmer to keep content and styling separate. That way we can keep our files simple and also allow for different webpages to have the same styling! Imagine if every time the New York Times had to publish an article they had to re-do the look of the entire newspaper!

What can CSS change?

  • Font type
  • Text sizes
  • Backgrounds
  • Borders
  • Page Layout
  • Margins
  • Image sizes
  • Rounding edges
  • LOTS MORE...

A syntax primer

CSS syntax is pretty straightforward!

The selector chooses the element whose formatting you want to change. It corresponds to the html tag of which you want to change the styling.
The declaration is how you're changing the element -- whether it's changing font size, background color, or position.

Linking CSS and HTML

You need to create a CSS stylesheet and link to it in your HTML file.
Your link should be in the 'head' section of the html and should look something like this:

<link rel="stylesheet" type="text/css" href="codealong.css">

What does the "cascading" in CSS mean?

If we look at the HTML of a page, we can see that elements are embedded within other elements. For example, in this page the h1 header is embedded within the 'body' element of this page, which is embedded in the 'html' element.
Imagine a big family tree. Now, back to cascades: When we change the formatting of a specific element, that change will cascade down to all of the related elements.

Why classes and ids?

Sometimes we have to get more granular than what the tags in our html can provide us. For example, we only want to convert one of our h3 headings to a different color. If we change the css of h3s, the will all change.

So we add classes and/or ids. Classes and ids are markers that we put in our HTML so that we can reference them in our CSS stylesheets. An id can only be used once in a page, but a class can be used multiple times. In our stylesheets, ids are referenced with a '#' and classes are referenced with a '.'.
